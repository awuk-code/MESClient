按钮功能修改说明

一、整体修改目标

本次主要完成几个按钮的基础功能：

1. 导出报表按钮：
点击后导出当前页面当前表格内容。

2. 生产任务表格中的开工按钮：
点击后带着当前生产任务行数据跳转到工序站点页面，并开始该产品的工序。

3. 工序站点右侧保存按钮：
当前功能待定，先保留点击入口和 qDebug 调试信息。

4. 维修判定页面保存按钮：
先把情况判定内容保存到本地临时 JSON 文件。

5. 维修判定页面返回按钮：
返回维修站页面。

6. 维修判定页面提交按钮：
上传接口先空着，当前只保留 qDebug 和提示信息。

所有待后续接接口的位置都添加了注释，方便后期继续修改。


二、导出报表按钮

修改文件：ui/basepagewidget.h

1. 新增函数声明：BasePageWidget::exportCurrentTableToExcel

函数作用：
导出当前页面正在显示的 QTableView。


修改文件：ui/basepagewidget.cpp

1. 修改函数：BasePageWidget::setupSearchLayout

修改内容：
给默认的导出报表按钮设置属性：
buttonRole = export。

修改原因：
工序站点右侧也使用 m_exportBtn 变量作为“保存”按钮。
为了避免“保存”按钮误触发导出逻辑，用 buttonRole 区分按钮用途。

2. 修改函数：BasePageWidget::initConnect

修改内容：
如果 m_exportBtn 的 buttonRole 是 export，则连接到 exportCurrentTableToExcel。

修改效果：
生产任务页面、维修站页面中真正的“导出报表”按钮会触发导出。
工序站点“保存”按钮不会触发导出。

3. 新增函数：BasePageWidget::exportCurrentTableToExcel

函数逻辑：
获取当前 QStackedWidget 正在显示的页面。
判断当前页面是否是 QTableView。
如果不是表格，则提示当前页面没有可导出的表格。
如果是表格，则弹出文件保存对话框。
默认文件名为：报表_年月日_时分秒.xls。
将当前表格的可见列和当前过滤后的可见行写入文件。

导出格式：
使用 Excel 可以直接打开的 HTML 表格格式，文件扩展名为 .xls。

后续扩展说明：
如果后面引入 xlsx 库，只需要替换 exportCurrentTableToExcel 里面的写文件逻辑。
页面按钮连接和调用入口不用改。

五、生产任务表格开工按钮

修改文件：core/operationdelegate.h

1. 修改信号：OperationDelegate::sigStartClicked

保留原信号：
sigStartClicked(int row)

新增信号：
sigStartClicked(const QVariantMap& rowData)

新增原因：
原来只传 row 行号。
点击开工后，行状态会变成“已开工”，当前“未开工”页签可能马上过滤掉该行。
如果再用 row 去当前表格回查数据，可能取错或取不到。

现在点击开工时直接把当前行完整 rowData 传出去，更稳定。


修改文件：core/operationdelegate.cpp

1. 修改函数：OperationDelegate::paint

修改内容：
开工按钮颜色不再依赖“标签已打印”。
只要任务不是“已完工”，开工按钮显示绿色。
已完工任务显示灰色。

修改原因：
用户要求开工按钮点击后进入工序站点。
原来必须先标签打印才能开工，导致开工按钮看起来不可用。

2. 修改函数：OperationDelegate::editorEvent

修改内容：
点击开工按钮时，只限制“已完工”不能再次开工。

点击后执行：
rowData[FIELD_STARTED] = true。
rowData["status"] = "已开工"。
调用 updateRowData 写回模型。
发出 sigStartClicked(index.row())，保留旧调试逻辑。
发出 sigStartClicked(rowData)，给页面跳转使用。

重要注释：
函数中已添加注释：
如果后续需要恢复“必须先标签打印”的限制，可以在这里增加接口或状态判断。


修改文件：core/productiontaskmodel.cpp

1. 修改函数：ProductionTaskModel::initConnect

修改内容：
由于 OperationDelegate::sigStartClicked 现在有两个重载版本，原来的连接需要指定 int 版本。

使用：
QOverload<int>::of(&OperationDelegate::sigStartClicked)

作用：
保留模型内原有 qDebug 调试输出。


修改文件：core/productiontaskmodel.h

1. 移除 Q_OBJECT

修改原因：
ProductionTaskModel 当前没有自定义 signal/slot，不需要 Q_OBJECT。
加上 Q_OBJECT 后工程没有生成对应 moc 文件，会导致链接错误。


修改文件：ui/productiontaskpage.h

1. 新增函数声明：ProductionTaskPage::startProcessByProxyRow

当前作用：
保留兼容旧的 int row 信号入口。
现在内部只做 Q_UNUSED。

2. 新增函数声明：ProductionTaskPage::startProcessByRowData

函数作用：
接收 OperationDelegate 传出的完整生产任务行数据，并跳转到工序站点。


修改文件：ui/productiontaskpage.cpp

1. 修改函数：ProductionTaskPage::ProductionTaskPage

修改内容：
获取 ProductionTaskModel。
连接 OperationDelegate 的两个开工信号：
int row 版本连接到 startProcessByProxyRow。
QVariantMap rowData 版本连接到 startProcessByRowData。

注意：
ProductionTaskModel 没有 Q_OBJECT，所以这里使用 dynamic_cast，不使用 qobject_cast。

2. 新增函数：ProductionTaskPage::startProcessByProxyRow

当前逻辑：
只保留兼容入口，使用 Q_UNUSED(proxyRow)。

3. 新增函数：ProductionTaskPage::startProcessByRowData

函数逻辑：
判断当前表格存在。
判断 rowData 不为空。
输出 qDebug，打印 taskNo 和 productModel。
调用 NavigationManager::openPage(PageType::ProcessStation, rowData)。

重要注释：
函数中已添加注释：
如果后续开工前需要调用 MES 开工接口，可以在跳转前补充接口请求。
