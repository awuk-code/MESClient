当前版本页面导航与返工任务单修改说明

一、整体修改思路

本次修改的核心目标是把软件中已经完成的几个页面统一纳入页面导航体系，并为后续接口和权限接入留下明确入口。

修改前的问题：
1. NavigationManager 只是部分注册页面，没有真正参与维修判定、返工任务单等业务页面跳转。
2. 工序站点页面注册时 PageType 写错，导致 NavigationManager 无法正确识别工序站点页面。
3. 维修判定页面和返工任务单页面没有作为独立页面注册，页面跳转逻辑不统一。
4. SubHeaderWidget 只能显示固定文本，不能根据当前页面显示页面导航，也没有预留当前工序显示位置。
5. 返工任务单和工序站点页面结构相似，需要复用原来的工序站点页面组件，减少重复代码。

修改后的思路：
1. PageType 中补充维修判定和返工任务单页面类型。
2. NavigationManager 统一负责页面注册、页面打开和页面数据传递。
3. MainWindow 只负责创建页面、注册页面、响应 NavigationManager 的页面打开信号并切换 QStackedWidget。
4. RepairStationPage 中的异常处理单号、维修判定按钮、返工任务单号统一通过 NavigationManager 跳转。
5. SubHeaderWidget 增加页面导航显示函数，同时预留当前工序显示入口。
6. 返工任务单复用 ProcessStationPage，只通过 setReworkTaskMode 切换左侧显示结构。


二、修改文件：commFunc/NavPageType.h

1. 修改 PageType 枚举。

新增页面类型：
RepairJudge：维修判定页面。
ReworkTask：返工任务单页面。

修改目的：
让维修判定页面和返工任务单页面可以和生产任务、工序站点、维修站一样，通过统一的 PageType 进行注册和跳转。


三、修改文件：core/navigationmanager.h

1. 新增 setPageData 函数。

函数作用：
保存指定页面打开时携带的数据。

使用场景：
从维修站点击某一行进入维修判定页面时，需要把当前行数据传给维修判定页面显示。

2. 新增 pageData 函数。

函数作用：
根据 PageType 获取之前保存的页面数据。

使用场景：
MainWindow 在打开 RepairJudge 页面时，通过 pageData(PageType::RepairJudge) 取出维修站传过来的行数据，然后调用 RepairJudgePage::setRepairData。

3. 新增 openPage(PageType type, const QVariantMap& data) 函数。

函数作用：
打开页面的同时携带当前行数据。

修改目的：
避免页面跳转时还需要额外写一套数据传递逻辑，让 NavigationManager 同时负责“打开哪个页面”和“带什么数据过去”。

4. 新增成员变量 m_pageDataMap。

变量作用：
保存每个页面对应的临时数据。


四、修改文件：core/navigationmanager.cpp

1. 新增 NavigationManager::setPageData 函数。

具体修改：
将 PageType 和 QVariantMap 保存到 m_pageDataMap 中。
增加调试信息，方便确认页面数据是否写入。

2. 新增 NavigationManager::pageData 函数。

具体修改：
从 m_pageDataMap 中按 PageType 返回页面数据。

3. 新增 NavigationManager::openPage(PageType type, const QVariantMap& data) 函数。

具体修改：
先调用 setPageData 保存页面数据，再调用 openPage(type) 打开页面。

特殊处理：
如果当前页面已经是目标页面，也会重新发出 sigOpenPage(type) 信号。
这样可以解决已经在维修判定页面时，再点击另一条异常处理单，页面不刷新的问题。

六、修改文件：ui/mainwindow.cpp

1. 修改 MainWindow::initUI 函数。

具体修改：
创建并注册生产任务页面：
PageType::ProductionTask。

创建并注册工序站点页面：
PageType::ProcessStation。

修复问题：
之前工序站点页面错误注册成了 PageType::ProductionTask，现在已修正为 PageType::ProcessStation。

创建并注册维修站页面：
PageType::RepairStation。

创建并注册维修判定页面：
PageType::RepairJudge。

创建并注册返工任务单页面：
PageType::ReworkTask。

返工任务单页面处理：
使用 ProcessStationPage 创建页面，然后调用 setReworkTaskMode(true)，让页面显示返工任务单的高级权限布局。

2. 修改 MainWindow::initConnect 函数。

具体修改：
打开 NavigationManager 的 sigOpenPage 信号连接。

现在流程：
NavigationManager::openPage(PageType)
发出 sigOpenPage(PageType)
MainWindow::onOpenPage(PageType)
切换页面

3. 修改 MainWindow::onOpenPage 函数。

具体修改：
通过 NavigationManager::page(type) 获取目标页面。
如果目标页面是 RepairJudge，则先调用 m_pageRepairJudge->setRepairData，把维修站传过来的行数据传给维修判定页面。
然后调用 m_stack->setCurrentWidget(page) 切换页面。
最后调用 updateSubHeaderNavigation(type) 刷新顶部导航。

4. 新增 MainWindow::updateSubHeaderNavigation(PageType type) 函数。

函数作用：
根据 PageType 更新顶部 SubHeaderWidget 的页面导航。

当前显示规则：
ProductionTask：生产任务 -> 生产任务列表。
ProcessStation：工序站点，并显示当前工序占位。
RepairStation：维修站 -> 维修站列表。
RepairJudge：维修站 -> 维修判定。
ReworkTask：维修站 -> 返工任务单，并显示当前工序占位。

当前工序说明：
工序站点和返工任务单都预留了当前工序显示。
当前工序未来需要通过接口获取，接口返回后调用 m_subHeader->setCurrentProcessName(processName) 即可。

5. 保留 MainWindow::updateSubHeaderNavigation(int index) 函数。

函数作用：
左侧菜单直接切换生产任务、工序站点、维修站时，继续根据菜单 index 更新导航。

十五、当前页面跳转流程

1. 左侧菜单点击生产任务。
SideBarWidget 发出 sigPageChanged(1)。
MainWindow::onPageChanged 切换到生产任务页面。
SubHeaderWidget 显示生产任务导航。

2. 左侧菜单点击工序站点。
SideBarWidget 发出 sigPageChanged(2)。
MainWindow::onPageChanged 切换到工序站点页面。
SubHeaderWidget 显示工序站点导航，并显示当前工序占位。

3. 左侧菜单点击维修站。
SideBarWidget 发出 sigPageChanged(3)。
MainWindow::onPageChanged 切换到维修站页面。
SubHeaderWidget 显示维修站导航。

4. 维修站待处理页点击维修判定按钮。
RepairStationPage 获取当前选中行数据。
调用 NavigationManager::openPage(PageType::RepairJudge, rowData)。
MainWindow::onOpenPage 打开维修判定页面。
RepairJudgePage::setRepairData 显示当前异常品数据。
SubHeaderWidget 显示维修站 -> 维修判定。

5. 维修站点击异常处理单号。
流程同维修判定按钮。

6. 维修站待返工页点击返工任务单号。
RepairStationPage 调用 NavigationManager::openPage(PageType::ReworkTask, rowData)。
MainWindow::onOpenPage 打开返工任务单页面。
SubHeaderWidget 显示维修站 -> 返工任务单，并显示当前工序占位。


十六、当前工序接口预留说明

当前工序现在还没有真实接口数据。
已经预留的位置：
SubHeaderWidget::setCurrentProcessName。

后续接口返回当前工序后，只需要调用：
m_subHeader->setCurrentProcessName(processName);

工序站点页面和返工任务单页面都会显示当前工序。
当前没有数据时显示“--”。
