新增类：

class MatAdapter

对外函数：

static bool isCompound(const QJsonArray& data);
static bool isCompound(const QVector<QVariantMap>& data);

作用：
判断数据是否为复合数据，即是否包含 materials 数组结构。
主要用于上传前校验处理后的数据结构。


static QVector<QVariantMap> split(const QJsonArray& compoundData);
static QVector<QVariantMap> split(const QVector<QVariantMap>& compoundData);

作用：
把后台复合数据拆成表格分离行数据。
用于工序站点表格显示前处理后台返回数据。


static QJsonArray combine(const QVector<QVariantMap>& splitRows);

作用：
在没有后台原始数据时，把表格行直接组合为后台需要的复合 JSON。


static QJsonArray combine(const QJsonArray& originalData,
                          const QVector<QVariantMap>& splitRows);

作用：
真实上传时优先使用。
以后台原始数据为基础，把表格修改、补充的数据合并回去，避免丢失后台原有 ID、状态、时间、工序字段、物料字段。


私有函数：

static Opt opt();
static bool hasItems(const QVariantMap& row, const Opt& opt);
static QVector<QVariantMap> expandNestedRows(...);
static QVector<QVariantMap> expandFlatRows(...);
static void appendGroup(...);
static QVariantMap itemFromRow(...);
static QVariantMap stripMeta(...);

作用：
这些函数只服务于内部拆分、分组、聚合和清理元数据，对外不暴露。


三、tool/matadapter.cpp
----------------------

文件位置：

tool/matadapter.cpp

主要函数：

MatAdapter::isCompound(...)

判断是否存在 materials 数组字段。


MatAdapter::split(...)

如果是复合数据，调用 expandNestedRows 展开 materials。
如果是平铺数据，调用 expandFlatRows 按 productSN 分组。


MatAdapter::combine(const QVector<QVariantMap>& splitRows)

按 productSN 或 _groupKey 将表格行聚合成：

[
  {
    "productSN": "...",
    "materials": [...]
  }
]


MatAdapter::combine(const QJsonArray& originalData,
                    const QVector<QVariantMap>& splitRows)

以 originalData 为底合并表格修改。
父级已有字段会从表格首行补回父级。
物料字段会合并回 materials。
内部字段 _groupKey、_groupIndex、_groupFirst、_groupRowCount、_itemIndex 不上传。


MatAdapter::expandNestedRows(...)

把后台一对多复合结构展开为多行表格数据。


MatAdapter::expandFlatRows(...)

把后台一对一或平铺结构按产品 SN 分组，补充序号和分组元数据。


MatAdapter::appendGroup(...)

给表格行补充 rowNumber、_groupKey、_groupIndex、_groupFirst、_groupRowCount、_itemIndex。


MatAdapter::itemFromRow(...)

从表格行中提取物料字段，排除产品 SN、序号、materials 和内部元数据。


MatAdapter::stripMeta(...)

上传前清理内部元数据。

八、tool/clientinfo.cpp
----------------------

文件位置：

tool/clientinfo.cpp

主要函数：

ClientInfo::ip()

返回真实网卡 IPv4。


ClientInfo::mac()

返回真实网卡 MAC。


ClientInfo::info()

返回 clientIP、clientMAC、hostName。


ClientInfo::primaryNetCard()

遍历本机网卡，选择真实物理网卡。


ClientInfo::isRealNetCard(...)

判断是否是真实网卡。
要求：

1. 网卡启用。
2. 网卡正在运行。
3. 非回环。
4. 非点对点。
5. MAC 有效。
6. 有 IPv4。
7. 名称不是虚拟网卡、VPN、Docker、WSL、蓝牙等干扰项。


ClientInfo::isFallbackNetCard(...)

兜底判断可用网卡。
当前也会过滤虚拟网卡，避免误上传干扰项。


匿名命名空间辅助函数：

isInvalidMac(...)

过滤空 MAC、00:00:00:00:00:00、FF:FF:FF:FF:FF:FF。


isInvalidIPv4(...)

过滤：

127.*
0.0.0.0
169.254.*
224.* 及以上
回环地址
空地址


isVirtualInterfaceName(...)

二十一、当前设计结论
--------------------

1. 一对多物料数据不建议直接在表格上传时重新造 JSON。
   更合适的做法是以后台原始数据为底，把表格修改和状态数据补回去。

2. MatAdapter 当前已经提供：
   判断复合数据、拆分显示数据、组合上传数据、基于原始后台数据合并上传数据。

3. ClientInfo 当前尽量获取真实物理网卡 IP/MAC。
   如果现场电脑有特殊网卡环境，上线前需要在真实设备上验证。

4. Camera 当前只是工具接口和占位实现。
   没有高拍仪 SDK 的阶段，可以先使用本地图片选择模拟拍照。
   后续可接厂家软件或 SDK。

5. NG 异常登记的“选择照片”按钮已经支持普通点击选择照片、长按切换为打开相机。


二十二、后续建议
----------------

1. 确认高拍仪厂家软件 exe 路径。
2. 确认高拍仪软件是否支持命令行启动。
3. 确认高拍仪拍照保存目录。
4. 确认高拍仪照片命名规则。
5. 后续在 Camera::openApp() 和 Camera::latestPhoto() 中补真实实现。
6. 真正上传时，把 PASS、NG、完成时间、工时等状态合并进 MatAdapter 上传数据。
7. 在真实现场电脑上验证 ClientInfo 获取到的 IP 和 MAC 是否符合预期。
